# 翻譯順序與原文取得規劃

> 哪部先做、原文在哪、信心如何、批次為何
> 啟動：2026-06-29

## 1. 排序原則

| 變數 | 權重 | 理由 |
|------|------|------|
| **語言熟悉度** | 高 | LLM 對該語言的閱讀 → 翻譯可靠度 |
| **資料可取得性** | 高 | 原文資料庫是否完整、版本是否單一 |
| **長度適中** | 中 | 先短後長，先建立工作流 |
| **完整性** | 中 | 短典可一次做完，長典要分批 |
| **宗教重要性** | 中 | 每個宗教的核心先做 |
| **翻譯歧義度** | 低 | 太歧義的反而不利於 demo，先做明確的 |

## 4. 各經典原文來源（已預查）

- **主版本**：王弼本（魏晉）
- **校勘來源**：
  - https://ctext.org/dao-jing （中文完整）
  - 郭店楚簡本圖版（荊門博物館）
  - 馬王堆帛書甲乙本圖版（湖南省博物館）
- **抓取策略**：以 ctext.org 王弼本為主，註明異文處引用帛書本

## 6. 工作流驗證步驟

每批啟動前：
1. 確認原文來源可用（URL 仍活躍）
2. 確認版本選定（含校勘優先序）
3. 測試小樣：取首段翻譯，看 LLM 對該語言 → 繁中白話品質
4. 確認交叉驗證譯者清單（≥2 譯者）
5. 確認 SOP 適用（若發現新問題，更新 methodology/）

## 7. 建議啟動

**先做批次 1：道德經 + 心經**

理由：
- 道德經：建立古典漢語完整 SOP（流程基準）
- 心經：建立梵文 + 漢譯對照 SOP（多語種基準）
- 兩個做完後，工作流完整驗證，再做批次 2

**等 user 確認後啟動抓原文 + 翻譯**
