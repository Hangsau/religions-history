# 經典版本問題 SOP

> 當「原文」不是單一語言 / 單一抄本時，怎麼處理
> 啟動：2026-06-29

## 1. 核心原則

### 1.1 第一原則：**拿最早版本**

- 跨語言版本比較時，**時間最早的**作為主要工作文本
- 不是「原文語言」，是「最早記錄」

### 1.2 第二原則：**原文 ≠ 原語言**

- 對許多佛教典籍，「原語言」可能是漢文（中亞漢化佛教圈產物）
- 對《六祖壇經》《大乘起信論》等 — 漢文即「原文」
- 「原文」是歷史 / 語言事實，不是規範要求

### 1.3 第三原則：**多語拼湊**

- 梵文 + 漢譯 + 藏譯 + 于闐文 + 龜茲文 + 粟特文 + 蒙文 + 西夏文 等
- 每一語言版本代表一個傳播路線
- 差異處是「演化痕跡」，可推回早期形式

## 2. 版本問題分類

### 2.1 A 類：原語言有古抄本（無問題）

例：《梨俱吠陀》、古蘭經（早期抄本 vs 奥斯曼定本）、舊約（死海古卷 vs 馬索拉）、新約（Papyrus 46 等）

→ 用最古抄本作工作文本

### 2.2 B 類：原語言抄本晚於翻譯（需說明）

例：心經、金剛經、阿彌陀經、楞嚴經

→ **取最早完整翻譯作工作文本**，標明「原語言抄本晚 X 年」

### 2.3 C 類：純漢文 / 純藏文（無原語言）

例：《大乘起信論》《六祖壇經》《阿毗達磨俱舍論》（真諦譯，但中國學者可能編）

→ **漢文 / 藏文即原文**，按該語言版本處理

### 2.4 D 類：版本繁多，無定本

例：《道德經》（多版本）、《奧義書》（108 部異本）、《往世書》（18 大 + 18 小）

→ 選定主要版本（多版本共識最強的）

### 2.5 E 類：口述傳統，無固定文本

例：《梨俱吠陀》早期、非洲傳統、北美原住民、波里尼西亞

→ 取學界標準文字化版本 + 多族群訪談對照

## 3. 處理流程

每部經典啟動翻譯前：

```
Step 1：盤點所有現存語言版本（時代先後）
Step 2：判定類型（A / B / C / D / E）
Step 3：選定「工作文本」
Step 4：拼湊多語版本差異（蛛絲馬跡）
Step 5：寫版本說明文件（translations/[經典]/00-version.md）
Step 6：AI 翻譯工作文本
Step 7：交叉驗證用其他語言版本
```

## 4. 寫作格式（每經典版本檔）

每個經典目錄下 `00-version.md`：

```markdown

|------|------|------|------------|------|
| [...] | [...] | [...] | [...] | [...] |

## 2. 學界爭論

（X 派 vs Y 派）

## 3. 蛛絲馬跡（多語對照）

| 段落 | 鳩摩羅什 | 玄奘 | 梵文 Gilgit | 梵文 Schøyen | 藏文 |

**工作文本**：[...]

（明確標出哪些無法確定）
```

## 5. 總攬位置

所有「有版本問題」的經典在 `00-overview/version-issues.md` 集中索引：

- 該表讓 user 一眼看到哪些經典有版本問題
- 點進各 .md 看詳細

## 6. 反例（不要做）

- ❌ 假裝原語言版本有定本，事實上沒有
- ❌ 把漢譯當原文又不標明
- ❌ 只看單一語言版本，不做多語對照
- ❌ 跳過學界爭論直接做翻譯
- ❌ 不交代「為什麼用這個版本」
